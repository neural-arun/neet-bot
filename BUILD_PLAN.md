# Build Plan — NEET Biology Telegram Bot

## Architecture

```
Telegram App ← → python-telegram-bot (polling)
                        ↓
                   SQLite (user progress, sessions)
                        ↓
              questions_dataset.json (32 chapters, 8000 Qs)
                        ↓
              Gemini API → ReportLab → PDF report
```
`
## Project Structure

```
neet_bot/
├── main.py              # Bot entry point (polling loop)
├── config.py            # Env vars: TELEGRAM_TOKEN, GEMINI_API_KEY
├── db.py                # SQLite schema + CRUD
├── questions.py         # Load JSON, filter by chapter, pick random N
├── bot_handlers.py      # All bot commands & callbacks
├── ai_analyzer.py       # Call Gemini for wrong-answer analysis
├── pdf_report.py        # ReportLab → PDF from analysis
├── data/
│   └── questions_dataset.json   # Symlink/copy
├── requirements.txt
└── Dockerfile
```

## Phase 1 — Database (SQLite)

### Tables

| Table | Columns | Purpose |
|-------|---------|---------|
| `users` | user_id, username, created_at | Track who's used the bot |
| `sessions` | id, user_id, chapter, total_q, answered_q, correct_q, status (active/done), created_at | One per test attempt |
| `answers` | id, session_id, q_index, question_text, user_answer, correct_answer, is_correct | Per-question record |

Use WAL mode for concurrent safety.

## Phase 2 — Question System

- Load `questions_dataset.json` into memory at startup (few MB)
- `get_random_questions(chapter_name, count=N)` → picks N random questions from that chapter
- `get_chapter_list()` → returns sorted list of 32 chapters

## Phase 3 — Telegram Bot Flow

### Commands & Callbacks

| Step | Trigger | Action |
|------|---------|--------|
| `/start` | Any message | Show welcome + 32 chapter buttons (paginated 8/page) |
| `chapter_{name}` | Callback | Store chapter, show count buttons: [10] [15] [25] [45] |
| `count_{N}` | Callback | Create session in SQLite, send Q1 |
| `answer_{idx}_{opt}` | Callback | Record answer, highlight selected option |
| `nav_prev` / `nav_next` | Callback | Show previous/next question (preserve answers) |
| `nav_submit` | Callback | Confirm dialog → score summary → trigger AI |
| `confirm_submit` | Callback | Finalize, calculate results, start AI analysis |

### Question Message Layout

Sent as a regular message (not a Telegram poll):

```
Question 3/25

At which stage of HIV infection does the patient start showing symptoms of AIDS?

[A] ● When CD4 count falls below 200 cells/μL
[B]   When CD4 count rises above 500 cells/μL
[C]   When viral load is undetectable
[D]   Immediately after infection

◀ Prev    [Submit]    Next ▶
```

- Tapping an option edits the message to show the selection (mark with ●)
- User can change answer by tapping another option before moving on
- Correct answer is shown only after Submit

## Phase 4 — AI Analysis (Gemini)

Prompt sent to Gemini 2.0 Flash (free tier):

```
You are a NEET Biology tutor. Analyze these mistakes:
- For each wrong question, explain WHY the student's choice is wrong
- Give a one-line memory tip for the correct answer
- Identify which topics/chapters are weak
- Suggest 2-3 specific topics to revise

Student scored {correct}/{total} on {chapter}.

Wrong answers:
Q1: {question text}
  Student chose: {wrong answer}
  Correct answer: {correct answer}
...
```

Response → structured text with sections: Score Summary, Mistake Analysis, Weak Areas, Revision Plan.

---

## Phase 5 — PDF Report (ReportLab)

PDF contents:
1. **Header**: "NEET Biology Test Report" + date
2. **Score**: X/Y correct (percentage)
3. **Chapter**: name tested
4. **Wrong Questions Table**: question, your answer, correct answer
5. **AI Analysis**: full text from Gemini (weak areas, tips)
6. **Footer**: "Generated by NEET Bot"

Generated as bytes → sent via `bot.send_document()` as `.pdf`

## Key Design Decisions

| Decision | Choice | Why |
|----------|--------|-----|
| Bot mode | **Polling** (not webhook) | No public URL needed → easier free hosting |
| Question display | **Regular message + inline keyboard** | Telegram quiz polls can't be edited or have prev/next |
| Answer reveal | **After Submit only** | Simulates real test conditions |
| State storage | **SQLite + per-user dict cache** | Survives bot restart, minimal complexity |
| Chapter nav | **8 chapters per page** | 32 chapters fit in 4 pages of inline buttons |
| Gemini model | **Gemini 2.0 Flash** | Free tier (60 req/min, 1500/day) |
